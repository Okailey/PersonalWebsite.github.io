WORDLE


INTRODUCTION


For this project, I teamed up with a partner to complete it. The task was both simple and complex: creating a Wordle game that resembles the New York Times version.
In object-oriented design, when building a piece of software, there are a few ways to begin:

• By examining a description of the desired software, called a requirements document, or a description of the physical system.
• By reverse engineering an existing implementation.



PROCESS



For the first part of this task, we had to identify classes and create a UML diagram, which would help with delegating some specific classes to the MVC (Model-View-Controller) architecture.


Initially, some classes were well-placed in terms of their architecture, but there was a challenge in figuring out how the model and controller would communicate without the view.

I was in charge of the view component and the controller component of the task. My job was to create the UI for the Wordle game and to make it as similar as possible to the New York Times version. I had learned a bit about JComponents and graphics for Java in Eclipse, so it provided some useful references. I initially used JButtons for the keyboard outline and display, but it proved to be a bit difficult because I could not get the color to change to a gray undertone, which was the minimum needed for the keyboard to look similar.

I moved on to graphics components, which was a bit more challenging due to the precision needed for each character box. I achieved a good layout, but it was not exactly like the New York Times version. It was difficult to manage the constraints for the keyboard graphics to ensure that when a letter was clicked, it did not go out of bounds or display another letter. After some adjustments, I managed to keep everything within the constraints of the overall keyboard rectangle.





This is what my view for the keyboard layout looks like above.







This is the New York Times keyboard layout. I opted for ASCII keys for the Enter and Backspace instead.


The next thing I planned was to draw the grid layout so that words would appear when a key on the keyboard is pressed. I used graphics to draw the grid, and it turned out well, resembling the one from the New York Times.


The next step was to ensure that once the game started and a player clicked on a keyboard key, the letter appeared on the grid, and when the Backspace key was clicked, the current letter was deleted. It was fairly easy to manage the appearance of the letter on the grid once it was pressed, as well as the Backspace key. The challenge was with the Enter key and the validation of the guess through colors.

My partner worked mainly on the model, where the game's data and logic are managed. The model handles the game state, including the target word, guesses, and the validation of those guesses. When the Enter key was pressed, a word that was not in the provided dictionary was still considered valid, which should not have been the case. After a few days of work, we resolved this issue. However, a new problem arose: testing with a known answer showed that the colors were not updating correctly.

For instance, if our target word was "PLACE" and we started with "PLAIN", the grid updated correctly the first time, where "P", "L", and "A" were green, and "I" and "N" were red. On the second attempt, if we guessed "PLACK", "P" and "L" turned yellow, indicating they were in the wrong position but part of the target word, which was incorrect. We had to adjust the logic for guessing and the UI for grid display. It took time to coordinate everything, but it worked well for most words, though some minor issues remained with color updates on the grid and keyboard.

The next feature I wanted to add was a text box displaying information about a win or indicating that you should try again after using all your guesses, as well as displaying the target word. I successfully incorporated this into the game logic, and it turned out quite well.

CONCLUSION



For this project, I have learned some new programming techniques (graphics and JComponents in Java), how to identify the necessary classes before coding, how to plan a software project, and how to work effectively in a team. These new skills will surely boost my morale to create more projects using this format and what I have learned.
